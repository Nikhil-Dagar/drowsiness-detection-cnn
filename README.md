# 😴 Driver Drowsiness Detection — CNN + Yawn Detection

A real-time drowsiness detection system that goes beyond simple eye-aspect-ratio
thresholds: a trained **CNN classifies eye state** (Open/Closed) per frame,
combined with **Mouth Aspect Ratio (MAR)** for yawn detection, live **event
logging**, and a **Streamlit dashboard** for demoing it.

## 🔍 Overview

Most beginner drowsiness-detection projects (including the classic PyImageSearch
tutorial this space is built on) use a single geometric rule: compute the Eye
Aspect Ratio (EAR) from facial landmarks and raise an alert once it drops below
a fixed threshold for N frames. It works, but it's brittle — EAR is sensitive to
camera angle, lighting, and per-person eye shape, and it only tracks eye closure.

This project keeps EAR as a lightweight cross-check but replaces it as the
*primary* signal with a **trained CNN eye-state classifier**, and adds a second,
independent signal — **yawning** — via Mouth Aspect Ratio. Every drowsy/yawn
event is timestamped and logged, and the whole thing is wrapped in a live
Streamlit dashboard.

| | Baseline (EAR-only) | This project |
|---|---|---|
| Eye-state decision | Fixed geometric threshold | Trained CNN classifier (+ EAR cross-check) |
| Signals used | Eye closure only | Eye closure **+** yawning (MAR) |
| Event history | None | Timestamped CSV log |
| Interface | Terminal / OpenCV window only | OpenCV window **+** Streamlit dashboard |
| Tunability | Hardcoded threshold | Live sliders for all thresholds |

## ✨ Features

- **CNN eye-state classifier** — trained on labeled Open/Closed eye crops, not a fixed geometric rule
- **Yawn detection** via Mouth Aspect Ratio (MAR), independent of the eye signal
- **Event logging** — every drowsy/yawn start & end timestamped to `logs/event_log.csv`
- **Repeating, non-blocking audio alarm** — replays every few seconds while drowsiness persists (not just once), runs on a background thread so the video feed never stutters
- **Streamlit dashboard** — live video feed, real-time status, adjustable thresholds, session log table
- **Configurable thresholds** for eye-closed probability, yawn MAR, consecutive-frame counts, and alarm repeat interval

## 🛠️ Tech Stack

- **OpenCV** — video capture, image preprocessing, drawing overlays
- **dlib** — 68-point facial landmark detection (pretrained model)
- **TensorFlow / Keras** — CNN training and inference for eye-state classification
- **Streamlit** — web dashboard for the live demo
- **scipy / numpy / pandas** — geometry calculations and log handling

## 🚀 Setup

### 1. Clone and set up the environment

```bash
git clone https://github.com/Nikhil-Dagar/drowsiness-detection-cnn.git
cd drowsiness-detection-cnn
python -m venv venv
venv\Scripts\activate        # Windows (cmd)
```

### 2. Install dlib first (Windows)

`dlib` needs to compile from source (CMake + Visual Studio Build Tools), which
fails on most machines out of the box. Easiest fix on Windows: grab a prebuilt
wheel matching your exact Python version from
[z-mahmud22/Dlib_Windows_Python3.x](https://github.com/z-mahmud22/Dlib_Windows_Python3.x)
(covers Python 3.7–3.14) and install it directly:

```bash
pip install dlib-<version>-cp313-cp313-win_amd64.whl   # match your Python version
```

(This is an unofficial community build — use CMake + Visual Studio Build Tools
instead if you'd rather build from source / official channels.)

### 3. Install the rest of the dependencies

```bash
pip install -r requirements.txt
```

### 4. Get the facial landmark model

Not included in this repo (it's ~95MB — over git's comfort zone). Download it
and place it in `models/`:

```
http://dlib.net/files/shape_predictor_68_face_landmarks.dat.bz2
```

Extract the `.bz2` and place `shape_predictor_68_face_landmarks.dat` inside `models/`.

### 5. Get an eye-state dataset and train the CNN

Any labeled Open/Closed eye dataset works — for example the **MRL Eye Dataset**
or Kaggle's **Driver Drowsiness Dataset**. If your download has a flat
`Open/` and `Closed/` folder structure, split it first:

```bash
python src/data_prep.py --src raw_dataset --dst dataset --val_split 0.2
python src/train_model.py --data_dir dataset --epochs 20
```

This saves the trained model to `models/eye_state_cnn.keras`.

### 6. Run it

**OpenCV window:**
```bash
python src/detect.py
```

**Streamlit dashboard:**
```bash
streamlit run app.py
```

Press `q` to quit the OpenCV window, or untick "Start Detection" in the
Streamlit sidebar.

## 🙏 Inspiration

The EAR/MAR landmark-geometry technique is based on the classic
[PyImageSearch drowsiness detection approach](https://www.pyimagesearch.com/2017/05/08/drowsiness-detection-opencv/)
by Adrian Rosebrock. This project extends that foundation with a trained CNN
eye-state classifier, yawn detection, event logging, and a Streamlit dashboard
on top of it.
